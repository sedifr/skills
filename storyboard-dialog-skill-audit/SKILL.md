# 目标

将零散的分镜对话行为整理为可复用的能力画像，帮助用户看见强项、短板和下一步训练动作。

# 执行流程

1. 识别复盘对象
- 先判断用户要的是“系统工具技能”还是“用户个人能力技能”。
- 若语义不清，优先按“用户个人能力技能”处理。

2. 抽取证据
- 仅基于当前会话内容提炼能力，不臆造未发生行为。
- 每个能力点都附对话证据（用户原话或关键行为）。

3. 能力归类
- 优先归入以下维度：需求澄清、目标对齐、镜头语言意识、节奏控制、审片标准、协作反馈质量。
- 若维度不适配，可新增 1-2 个贴合维度。

4. 评估成熟度
- 按三档标记：已稳定、已出现但不稳定、尚未体现。
- 给出判断理由，避免只有结论没有依据。

5. 生成行动建议
- 每次给 3 条以内可立即执行的训练动作。
- 每条建议都要可验证（例如“下一轮对话先给镜头范围和通过标准”）。

# 输出格式

按以下结构输出，保持简洁：

## 已体现技能
- 技能名：一句解释 + 证据

## 待加强技能
- 技能名：缺口说明 + 具体风险

## 下一轮可执行动作
- 动作 1
- 动作 2
- 动作 3

# 质量门槛

- 不把泛泛夸奖当作技能结论。
- 不输出超过用户当前上下文的信息。
- 优先给“对下一次分镜协作马上有用”的建议。
